1. 概述

B站推荐页使用内部代号 Pegasus（飞马） 的推荐系统。该系统负责首页的信息流推荐，根据用户的观看历史、兴趣偏好等数据，通过个性化推荐算法返回视频/直播/文章等内容卡片。

核心 API 端点为 /x/v2/feed/index，使用 GET 方法请求，Base URL 为 https://app.bilibili.com。

ℹ️ 响应格式

API 响应为 JSON 格式，外层包裹 GeneralResponse（含 code / message / data），data 部分即 PegasusFeedResponse。

2. API 端点

2.1 推荐信息流（主接口）

GET https://app.bilibili.com/x/v2/feed/index

获取推荐信息流数据，支持下拉刷新、上拉加载更多、Tab 切换等场景。

返回类型：GeneralResponse<PegasusFeedResponse>

2.2 推荐 Tab 信息

GET https://app.bilibili.com/x/v2/feed/index/tab

获取推荐页顶部运营 Tab 配置。

参数类型说明
tab_idStringTab 标识（可选）
idString运营 ID（可选）

2.3 负反馈接口

GET https://app.bilibili.com/x/v2/feed/nf

获取对某条推荐的"不感兴趣"反馈选项，用于展示负反馈弹窗。

参数类型说明
gotoString卡片跳转类型（如 av, live）
idlong卡片 ID
up_idlongUP主 ID
up_nameStringUP主名称
jumpgotoString跳转类型
video_modelong视频模式
columnlong列号
tnameString分区名称
fromlong来源

8. 鉴权与签名

8.1 签名机制

所有请求需经过 LibBili.signQuery() 签名。签名流程：

  1. 收集所有请求参数（URL Query + 公共参数 + 拦截器注入参数）
  2. 按 Key 字母序排列参数
  3. 调用 LibBili.signQuery(map) 生成签名
  4. 签名结果附加在 URL 的 sign 参数中

8.2 鉴权方式

  • 登录状态：通过 access_key 参数传递用户令牌
  • 游客访问：无 access_key 时使用 GuestId Header
  • 设备标识：通过 Buvid Header 和 Device-ID Header 标识设备
⚠️ 签名算法

LibBili.signQuery() 是 native 方法（SO 库实现），签名算法涉及到 appkey/appsecret，不同客户端版本使用不同的密钥。逆向时需要从 libbili.so 中提取签名逻辑。

9. 注意事项

⚠️ 风控与封禁

B站对 API 调用有严格的风控机制。异常调用模式（高频请求、缺少必要 Header、签名错误等）可能触发风控，导致 IP 封禁或账号限制。

ℹ️ 分页机制

推荐信息流使用 游标分页（而非页码分页）。首次请求 idx=0，后续请求使用上次响应中最后一条 item 的 idx 值。通过 ver 字段实现 Feed 版本追踪。

✅ 推荐追踪

每个 item 的 trackId 字段用于推荐效果追踪。点击/曝光上报时必须携带此字段，否则推荐质量会下降。

关键注意点汇总

关注点说明
请求签名 必须正确实现 LibBili.signQuery() 签名，缺少签名或签名错误会被拒绝
设备标识 Buvid、Device-ID、fp_local、fp_remote 等设备指纹缺一不可
User-Agent 必须使用 Bilibili APP 的标准 UA 格式，异常 UA 可能被拦截
游标分页 使用 idx 游标分页，不要使用 offset/limit 模式
广告卡片 响应中可能包含广告卡片（is_ad=true），需根据业务需要过滤或展示
卡片类型多样 响应中 items 可包含多种 cardType，需根据 card_type 字段分发到不同的渲染逻辑
暗色模式 颜色相关字段通常提供日间/夜间两套值（如 text_color / text_color_night）
刷新类型 不同刷新场景（下拉/Tab切换/上拉加载）使用不同的 flush 值，影响推荐策略
负反馈机制 提供 /x/v2/feed/nf 接口获取负反馈选项，对推荐系统优化很重要
Protobuf 并存 部分新版接口使用 Protobuf 格式（com.bapis.bilibili.app.card.v1），与 JSON 格式并存
API 版本 本文档基于特定版本的 TV 客户端分析，B站 API 可能随时变更
